# 监理审查项目实施计划 V2

## 1. 目标
- 在 V1 可用基础上，持续降低误报/漏报，提升日志审查准确性与可解释性。
- 形成“规则迭代 -> 回归验证 -> 推送门禁”的稳定工程化闭环。
- 输出可直接落地的审查结果：问题明细、完整日志草案、可归档报告。

## 2. 本阶段范围（V2）
- 规则精调：重点优化人员项、施工部位、高风险作业、安全人员体现。
- 样本体系：持续扩充真实样本并固化为回归用例。
- 输出体验：提升“建议替换后的完整日志（草案）”可用性。

## 3. 非范围（当前不做）
- 引入复杂机器学习模型或外部在线推理服务。
- 多租户权限系统与企业级用户管理。
- 大规模前后端重构。

## 4. 验收标准
- 回归集通过率保持 100%（当前门禁基线）。
- 新增真实样本回归用例不少于 20 条，覆盖主要误报/漏报场景。
- 对同类问题给出统一、可执行的完善建议文本（非碎片化建议）。
- 每次 push 自动触发回归，失败时能阻断推送。

## 5. 任务清单

### P0（当前优先）
- [ ] 将“人员必填项”规则永久固化到 `rules_v1.xlsx`（当前部分为运行时覆盖逻辑）。
- [ ] 补充 10 条真实样本并完成 `expected` 标注，更新回归集。
- [ ] 优化“完整日志草案”模板，统一输出结构：
  - 日期/天气
  - 施工部位
  - 作业内容
  - 现场人员（管理人员X名、工人X名）
  - 问题与整改
  - 复查与结论

### P1（第二优先）
- [ ] 增加“规则变更说明”记录模板（每次调规则记录影响面）。
- [ ] 分流清单自动化：从回归报告生成“TOP误报/漏报待修项”。

### P2（后续）

## 6. 迭代规则
- 单次迭代只做 1 个 P0 任务，完成后再切下一个。
- 每次改规则必须跑回归：`powershell -ExecutionPolicy Bypass -File .\scripts\run_regression.ps1`
- 回归通过后再提交；提交信息建议以 `v2:` 前缀。
- 旧 plan（`docs/implementation-plan.md`）仅保留历史，不再追加新任务。
